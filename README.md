# Trove

영화, 게임, 음악, 스니커즈와 라이프스타일 드롭처럼 기다리는 출시일을 한곳에 모으는 모바일 우선 웹 MVP입니다. 제공된 Figma의 `취향 선택`, `홈`, `캘린더`, `상세` 네 화면을 기준으로 구현했습니다.

## 실행

```bash
npm install
npm run dev
```

기본 개발 주소는 `http://127.0.0.1:4173`입니다.

```bash
npm run check
npm run build
```

## 구현된 흐름

- 관심사 검색·선택 및 선택값 저장
- 관심사 우선순위를 반영한 홈 추천
- 카테고리 필터와 가로 카드 탐색
- 좋아요 저장·해제
- 상세 이미지 캐러셀과 공유
- D-7, D-1, 당일 알림 프리셋 저장
- 월/주 캘린더, 날짜 선택, 일정 목록
- 알림·준비 중·설정 바텀시트
- 새로고침 후에도 유지되는 로컬 상태
- 360×780 모바일 레이아웃과 430px 데스크톱 앱 셸

## 데이터 연결 지점

현재 데이터는 [`src/data/mock.ts`](./src/data/mock.ts)의 `ReleaseItem[]`에 분리되어 있습니다. 향후 헤르메스 XLSX → staging DB → 승인된 `release_items` API 흐름을 붙일 때 페이지 컴포넌트를 바꾸지 않고 이 데이터 공급부만 교체할 수 있습니다.

이번 프로토타입에는 회원 인증, 실제 DB, XLSX 운영 콘솔, 예약 웹푸시와 결제는 포함하지 않았습니다. 알림 설정은 제품 흐름을 검증하기 위해 브라우저 로컬 저장소에 보관합니다.
